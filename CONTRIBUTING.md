## âœï¸ Style Guidelines

### **Writing:**
- Clear, accessible language
- Academic tone for research
- Empathetic tone for personal stories
- No excessive jargon

### **Markdown:**
- Use headers appropriately (`#`, `##`, `###`)
- Include table of contents for long docs
- Use code blocks for code/examples
- Add horizontal rules for section breaks

### **File Naming:**
- Lowercase with hyphens: `case-study-chatgpt.md`
- Descriptive names
- Include dates when relevant

### **Commits:**
- Clear, descriptive messages
- Format: `[TYPE] Brief description`
- Types: `ADD`, `FIX`, `UPDATE`, `DOCS`
- Example: `ADD case study for Claude AI testing`
